# Тест-кейсы: Boards API

## TC-BOARD-001 — Получение списка досок авторизованного пользователя

**Приоритет:** High

### Предусловие

Пользователь зарегистрирован в Trello.

В Postman создано и выбрано окружение `Trello API Environment` с валидными значениями:

* `base_url`;
* `api_key`;
* `api_token`.

### Шаги

1. Отправить GET-запрос:

   `{{base_url}}/members/me/boards`

2. Передать query-параметры:

   * `key={{api_key}}`;
   * `token={{api_token}}`;
   * `fields=name,url`.

3. Нажать `Send`.

### Ожидаемый результат

1. Сервер возвращает статус `200 OK`.
2. Ответ приходит в формате JSON.
3. Тело ответа является массивом.
4. Если у пользователя есть доски, каждый объект содержит поля:

   * `id`;
   * `name`;
   * `url`.
5. Если у пользователя нет досок, возвращается пустой массив `[]`.

### Постусловие

Данные пользователя и досок не изменены.

### Фактический результат

Сервер вернул статус `200 OK`.
Ответ получен в формате JSON-массива.
Объекты досок содержат поля `id`, `name` и `url`.

**Статус:** Passed

## TC-BOARD-006 — Создание доски с валидными данными

**Приоритет:** High

### Предусловие

Пользователь авторизован в Trello.

В Postman создано и выбрано окружение `Trello API Environment` с валидными значениями:

* `base_url`;
* `api_key`;
* `api_token`.

Доска с названием `QA API Test Board` отсутствует либо допускается создание доски с таким же названием.

### Шаги

1. Отправить POST-запрос:

   `{{base_url}}/boards`

2. Передать query-параметры:

   * `key={{api_key}}`;
   * `token={{api_token}}`;
   * `name=QA API Test Board`;
   * `defaultLists=false`.

3. Нажать `Send`.

### Ожидаемый результат

1. Сервер возвращает статус `200 OK`.
2. Ответ приходит в формате JSON.
3. В ответе присутствует непустое поле `id`.
4. Поле `name` содержит значение `QA API Test Board`.
5. Поле `closed` содержит значение `false`.
6. В ответе присутствуют поля `url` и `shortUrl` со ссылками на созданную доску.
7. Поле `prefs.permissionLevel` содержит уровень доступа к доске.
8. Новая доска отображается в интерфейсе Trello.
9. Доска создана без списков, поскольку параметр `defaultLists` передан со значением `false`.

### Постусловие

Создана доска `QA API Test Board`.

### Фактический результат

Сервер вернул статус `200 OK`.

Ответ получен в формате JSON и содержит:

* непустое поле `id`;
* `name: "QA API Test Board"`;
* `closed: false`;
* поля `url` и `shortUrl`;
* `prefs.permissionLevel: "private"`.

Доска успешно создана без списков.

**Статус:** Passed
