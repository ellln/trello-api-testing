# Тест-кейсы: Boards API

## TC-BOARD-001 — Получение списка досок авторизованного пользователя

**Приоритет:** High

### Предусловие

Пользователь зарегистрирован в Trello.

В Postman создано и выбрано окружение `Trello API Environment` с валидными значениями:

* `base_url`;
* `api_key`;
* `api_token`.

### Шаги

1. Отправить GET-запрос:

   `{{base_url}}/members/me/boards`

2. Передать query-параметры:

   * `key={{api_key}}`;
   * `token={{api_token}}`;
   * `fields=name,url`.

3. Нажать `Send`.

### Ожидаемый результат

1. Сервер возвращает статус `200 OK`.
2. Ответ приходит в формате JSON.
3. Тело ответа является массивом.
4. Если у пользователя есть доски, каждый объект содержит поля:

   * `id`;
   * `name`;
   * `url`.
5. Если у пользователя нет досок, возвращается пустой массив `[]`.

### Постусловие

Данные пользователя и досок не изменены.

### Фактический результат

Сервер вернул статус `200 OK`.
Ответ получен в формате JSON-массива.
Объекты досок содержат поля `id`, `name` и `url`.

**Статус:** Passed

## TC-BOARD-002 — Получение списка досок с невалидным API token

**Приоритет:** High

### Предусловие

В Postman создано и выбрано окружение `Trello API Environment` с валидными значениями:

* `base_url`;
* `api_key`.

### Шаги

1. Отправить GET-запрос:

   `{{base_url}}/members/me/boards`

2. Передать query-параметры:

   * `key={{api_key}}`;
   * `token=invalid_token`;
   * `fields=name,url`.

3. Нажать `Send`.

### Ожидаемый результат

1. Сервер возвращает статус `401 Unauthorized`.
2. В теле ответа содержится сообщение `invalid app token`.
3. Список досок пользователя не возвращается.

### Постусловие

Данные пользователя и досок не изменены.

### Фактический результат

Сервер вернул статус `401 Unauthorized`.
Тело ответа содержит сообщение `invalid app token`.
Список досок пользователя не был возвращён.

**Статус:** Passed

## TC-BOARD-003 — Получение списка досок без API token

**Приоритет:** High

### Предусловие

В Postman создано и выбрано окружение `Trello API Environment` с валидными значениями:

* `base_url`;
* `api_key`.

### Шаги

1. Отправить GET-запрос:

   `{{base_url}}/members/me/boards`

2. Передать query-параметры:

   * `key={{api_key}}`;
   * `fields=name,url`.

3. Не передавать параметр `token`.

4. Нажать `Send`.

### Ожидаемый результат

1. Сервер возвращает статус `400 Bad Request`.
2. В теле ответа содержится сообщение об отсутствии или недействительности токена авторизации.
3. Список досок пользователя не возвращается.

### Постусловие

Данные пользователя и досок не изменены.

### Фактический результат

Сервер вернул статус `400 Bad Request`.
В теле ответа содержится сообщение `invalid token`.
Список досок пользователя не был возвращён.

**Статус:** Passed
