# Тест-кейсы: Boards API

## TC-BOARD-001 — Получение списка досок авторизованного пользователя

**Приоритет:** High

### Предусловие

Пользователь зарегистрирован в Trello.  
В Postman создано и выбрано окружение `Trello API Environment` с валидными значениями:

- `base_url`
- `api_key`
- `api_token`

### Шаги

1. Отправить GET-запрос:

   `{{base_url}}/members/me/boards`

2. Передать query-параметры:

   - `key={{api_key}}`
   - `token={{api_token}}`
   - `fields=name,url`

### Ожидаемый результат

1. Сервер возвращает статус `200 OK`.
2. Ответ приходит в формате JSON.
3. Тело ответа является массивом.
4. Каждый объект доски содержит поля:
   - `id`
   - `name`
   - `url`
5. Если у пользователя нет досок, возвращается пустой массив `[]`.

### Постусловие

Данные пользователя и досок не изменены.


## TC-BOARD-002 — Получение списка досок с невалидным API token

**Название:** Получение списка досок с невалидным API token

**Приоритет:** High

**Предусловие:**
В Postman создано и выбрано окружение `Trello API Environment` с валидными значениями `base_url` и `api_key`.

**Шаги:**

1. Отправить GET-запрос:

   `{{base_url}}/members/me/boards`

2. Передать параметры:

   * `key={{api_key}}`
   * `token=388939949304`
   * `fields=name,url`

3. Нажать `Send`.

**Ожидаемый результат:**

1. Сервер возвращает статус `401 Unauthorized`.

2. В теле ответа содержится сообщение:

   `invalid app token`

3. Список досок пользователя не возвращается.

**Постусловие:**
Данные пользователя и досок не изменены.

**Фактический результат:**
Получен статус `401 Unauthorized`.
Тело ответа: `invalid app token`.

**Статус:** Passed
